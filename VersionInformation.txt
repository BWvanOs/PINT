Version history
New in version 0.5!
Image creation tool that complements MCD viewer
    -The tool uses the parameter table you created when denoising, so you can now use the thresholded/normalizated/denoised/transformed images as staining examples

open problems
    -If global normalization for a given channels fails it's not fed back to the user. It is stored in the resulting csv.
    -Winsorization floor value is not user adjustable.
    -Unable to pick own color for channels
    -No automated export function for all composit images


0.4.1
Added more comments to explaning the workings of the pipeline
bugfixes:
    -analysis.py main() would turn "Not a Number"/NaN" to "nan" and pd.NA would not pick up the missing values, leading to unexpected behaviour
    -If you added a stack of tiff where images >1 through n contained all channelnames from image 1, but channelsnames(1) != channelnames(!1) it would continue loading.
     it now take image 1 channels as ground truth and check for each subsequent imag of the channel names match exactly.
    -Tkinter scaling issue on Linux. File selection screen was TINY, but now it should be usable. Zenity is still the prefered one.
    -When the percentage of the staining was smaller than the winsorization filter, it would push the staining intensity into the background. Mitigation is that the staining is not set to 5 (not user adjustable yet)
     Background of the Hyperion(XTi) is 3~3.5 so setting it to 5 should fix this. Temporary fix untill a better method is implemented

open problems
    -If global normalization for a given channels fails it's not fed back to the user. It is stored in the resulting csv.
    -Winsorization floor value is not user adjustable.

0.4
New version switches to absolute thresholding as a default, relative thresholding still available (percentile based) but by default off
Analysis order still the same: Winsorization -> Thresholding -> Denoising -> Transformation -> Normalization, with all steps being optional
